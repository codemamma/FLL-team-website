Looking at the UNEARTHED mission layout and the limited space to move on the Robot Game table, we designed our robot to be compact, precise, and easy to control. We focused on accurate navigation, reliable attachments, and consistent performance to complete missions efficiently in tight spaces.

Robot Design

Our robot was designed to be compact, accurate, and reliable. We focused on wheel choice, motor placement, and modular attachments to reduce drift and improve consistency.

  • Compact base for tight turns
  • Motors mounted low for stability
  • Smaller wheels for higher accuracy

Attachments and Mechanisms

We designed multiple attachments to complete different missions efficiently. Each attachment was tested and improved through iteration.

  • Box Attachment: Push, pull, flick â†’ iteration removed bottom pegs
  • Excavator: More reliable for grabbing and digging
  • Passive Ramp: Saves time by not requiring motors

Programming and Logic

Our code focuses on accuracy, repeatability, and self-correction. We used gyro-based navigation and proportional turning to improve alignment and consistency.

  • Gyro reset before each run
  • Turning using yaw
  • Slower turns for accuracy
  • Auto-correct drive straight code for improved alignment
  • Proportional Left and Right Turn Code -- slows the robot as it approaches the target angle, increasing accuracy compared to non-proportional turns.

Strategy and Iterations

We estimated the number of moves required for each mission and prioritized missions with the highest points per move. We coded missions 6 and 7 first, followed by missions 5 and 10. After qualifiers, we optimized our code and strategy to improve speed and reliability.

  • Points per move strategy
  • Stopwatch used as a key practice tool
  • Practiced with partner teams
